# Changelog — MoreCheckmarks RU

## 2.1.0.7 (2026-06-27)

- **Fix:** «Показывать будущие уровни» = ВЫКЛ — убран жёсткий фильтр `IsHideoutAreaActionable`, из-за которого при выключенной опции вообще не показывались требования убежища. Теперь при OFF показывается только следующий уровень для всех зон (классическое поведение upstream).
- **Debug:** оставлено подробное логирование (вкл по умолчанию).

## 2.1.0.6 (2026-06-27)

- **Fix:** восстановлен поиск hideout-требований по текущим/будущим уровням (убрана лишняя фильтрация по `ad.NextStage`, которая могла полностью скрывать подсказки).
- **Debug:** добавлено подробное логирование расчёта hideout-подсказок (`[DEBUG] [GetNeeded]`, `[DEBUG] [Tooltip:Show]`, `[DEBUG] [Tooltip:SetTooltip]`) + настройки в F12 (`Логи hideout-подсказок`, `Логировать каждый тултип`).

## 2.1.0.5 (2026-06-27)

- **Fix:** «Показывать будущие уровни» ВЫКЛ → теперь отображаются **все** текущие доступные зоны ReadyTo*, где нужен предмет. Для каждой зоны показывается реальный прогресс: «Нужно для области: Освещение lvl2 (3/10)». Если предмет требуется для нескольких активных улучшений — показываются все.

## 2.1.0.4 (2026-06-27)

- **Fix:** жёлтая метка для **текущего** активного квеста (как в ванили; раньше подсказка была, цвет — нет)
- **Fix:** «Показывать будущие уровни» ВЫКЛ → только зоны ReadyTo* (стройка доступна сейчас), без Locked*

## 2.1.0.3 (2026-06-27)
